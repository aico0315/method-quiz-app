# 問題データ（data/questions.ts）

## ファイルの役割

問題の内容を全てここに書いています。データとロジックを分離することで、「問題を追加・修正したいとき」にこのファイルだけ触ればよくなります。

---

## データの形

```typescript
{
  id: "1",
  level: "junior",
  category: "配列メソッド",
  question: "配列の各要素を変換して新しい配列を作りたいとき",
  answer: ["map"],
  supplement: "arr.map(x => x * 2) のように使う。元の配列は変更しない非破壊メソッド。",
}
```

`Question` 型（types.ts で定義した形）に合わせてオブジェクトを並べているだけです。

## 問題を追加したいとき

このオブジェクトをコピーして `questions` 配列に追加するだけです。

```typescript
{
  id: "31",                     // 既存IDと重複しない番号
  level: "junior",              // "junior" | "middle" | "senior"
  category: "配列メソッド",      // 自由に設定
  question: "問題文をここに書く",
  answer: ["正解のメソッド名"],   // 複数正解があれば ["answer1", "answer2"]
  supplement: "使い方の補足説明",
}
```

---

## なぜ JavaScript ファイルに書くのか

- データベース（Supabase など）は「ユーザーがついてから」検討する方針
- 今はシンプルに `.ts` ファイルに直書きすることで、余計な複雑さなしに動かせる
- 問題を追加・修正するときもコードを読むだけでわかりやすい
