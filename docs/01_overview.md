# メソッド問題アプリ：全体像

## どんなアプリか

**「こういうことをやりたい」と思ったとき、咄嗟に使うメソッドが浮かぶ**ようになるための練習アプリです。

問題例：
> 配列の各要素を変換して新しい配列を作りたいとき → `map`

正しいメソッド名をテキストで入力して回答します。回答後は補足説明も表示されるので、知識を定着させながら進められます。

---

## 使っている技術

| 技術 | 役割 | ひと言 |
|---|---|---|
| **Vite** | 開発環境のビルドツール | ファイルを保存するたびにブラウザが即座に更新される |
| **React** | UI フレームワーク | 画面の部品（コンポーネント）を組み合わせてUIを作る |
| **TypeScript** | 言語 | JavaScript に「型」を追加して、ミスを事前に検出できる |
| **CSS Modules** | スタイリング | CSS のクラス名が他のファイルと衝突しない仕組み |

---

## 画面の流れ

```
メニュー画面（レベル選択）
  ↓ レベルを選ぶ
問題画面（問題表示 → 回答入力 → 正誤判定・補足表示）
  ↓ 次の問題へ（繰り返し）
クリア画面（スコア表示 → もう一度 or メニューへ）
```

3つの画面を1枚のページの中で切り替えています（SPA：シングルページアプリケーション）。

---

## ファイル構成

```
src/
├── main.tsx              # アプリの起動ファイル
├── App.tsx               # 全体の「司令塔」。画面の切り替えを管理する
├── types.ts              # TypeScript の型定義
├── index.css             # 全体に適用するスタイル
├── data/
│   └── questions.ts      # 問題データ（30問）
└── components/
    ├── MenuScreen.tsx     # メニュー画面
    ├── QuizScreen.tsx     # 問題・回答画面
    └── ClearScreen.tsx    # クリア画面
```

各コンポーネント（`.tsx`）には対応する `.module.css` ファイルがあり、見た目はそこに書いています。
